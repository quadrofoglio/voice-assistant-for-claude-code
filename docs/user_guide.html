🚀 Getting Started

First Launch

1
Navigate to dist\ inside the project folder and double-click VoiceAssistant.exe.
2
A small window opens and a blue icon appears in your system tray (bottom-right corner of the taskbar).
3
On first run, the app automatically configures Claude Code's Stop hook so responses are forwarded to the app. No manual setup needed.
4
To create a desktop shortcut, right-click VoiceAssistant.exe → Send to → Desktop (create shortcut).
Note: Make sure voices\ and piper_bin\ folders are in the same directory as VoiceAssistant.exe for offline TTS to work.

What the Main Window Shows

Status
Shows the current state: Idle, Recording…, or Transcribing…
Last Transcription
The most recent thing you said (after STT processed it).
Play Response
Reads Claude's last response aloud. Same as pressing Alt + R.
Mute
Silences TTS output without closing the app.
Settings
Opens the settings panel to configure hotkeys, voices, and modes.

🗣️ Speaking to Claude

Use push-to-talk to dictate your message instead of typing.

1
Hold Alt + Space — the status changes to Recording…
2
Speak your message clearly into your microphone.
3
Release the hotkey — the app transcribes your speech (status: Transcribing…).
4
A preview overlay appears at the bottom of your screen showing what was transcribed.
5
The text auto-sends after 2 seconds. Click Send to send immediately or Cancel to discard.
6
The text is delivered to your active window — either auto-typed (terminal mode) or copied to clipboard (clipboard mode).
Tip: In terminal mode, your text lands in the terminal but is not submitted — press Enter yourself to send it to Claude Code.

Speech Recognition Model

The app uses OpenAI Whisper running locally on your machine — no internet required for transcription. The default model is small, which balances accuracy and speed well. You can switch to tiny (faster) or medium (more accurate) in Settings.

🔊 Hearing Claude's Responses

When Claude Code finishes responding, the app automatically captures the response text and can read it aloud.

On-Demand Playback (default)

Press Alt + R at any time to hear Claude's last response read aloud. This is the default — you control when to listen.

Auto-Play

Enable Auto-play TTS in Settings → Output to have every response read automatically the moment Claude finishes. Useful when you're not looking at the screen.

What Gets Read

The app reads only the conversational prose — explanations, summaries, and answers. It automatically skips:

  • Code blocks (fenced with ``` or inline with backticks)
  • URLs and links
  • Markdown headers and formatting symbols

The surrounding explanation of what code does is read — so you hear the context without sitting through 50 lines of Python being read character by character.

Muting

Click Mute in the main window, or right-click the tray icon and select Mute TTS, to silence all audio output instantly.

Hotkey Format

Hotkeys use the keyboard library's syntax. Examples:

  • alt+space
  • ctrl+r
  • shift+f1
  • windows+v

Valid modifiers: alt, ctrl, shift, windows

Tip: Avoid hotkeys already used by your terminal or browser (e.g. ctrl+c, ctrl+v).

🗣️ TTS Engines

Edge TTS Online Free

Microsoft's neural voices. Sounds like a real person. Requires internet. Default engine.

Available voices include: Aria (US female), Guy (US male), Jenny (US female), Sonia (UK female), Ryan (UK male), Natasha (AU female), William (AU male), Neerja (IN female).

Piper TTS Offline Free

Neural voices that run fully on your machine. No internet needed. Slightly lower quality than Edge TTS but excellent for offline use.

Default voice: en_US-amy-medium (bundled, ~60MB)

To add more voices: Settings → Output → select Piper voice → click Download selected Piper voice.

How Fallback Works

If Edge TTS is selected but fails (e.g. no internet), the app automatically falls back to Piper if a voice is available. If neither works, TTS is silently skipped — you can still read the terminal.

📋 Input Modes

Terminal Mode (default)

Transcribed text is auto-typed into whatever window is currently focused. Designed for use with Claude Code CLI in a terminal.

After the text appears in your terminal, press Enter to submit it to Claude.

Clipboard Mode

Transcribed text is copied to your clipboard. Switch to this mode when using Claude.ai, ChatGPT, or any web-based AI chat. Then paste with Ctrl + V.

Switch between modes in Settings → Input → Input mode.

Tip: Keep it on terminal by default for Claude Code, and switch to clipboard when you open a browser chat.

👂 Wake Word

Instead of holding a hotkey, you can say a trigger phrase to start recording hands-free — like saying "Hey Google" or "Alexa".

Enable Wake Word

  1. Open Settings → Input
  2. Check Enable wake word
  3. Set your desired phrase in the Wake word field (default: hey claude)
  4. Click Save

Usage

Say your wake word clearly. The app starts recording. Speak your message. Pause — the app detects silence and stops recording automatically, then shows the transcription preview.

Note: Wake word detection uses always-on microphone listening and consumes slightly more CPU. Push-to-talk is recommended if you're on battery or in a noisy environment.

🔧 Troubleshooting

App doesn't respond to my hotkey

  • Make sure the app is running (check the system tray for the blue icon)
  • Try running as Administrator if another app is blocking global hotkeys
  • Check Settings to confirm the hotkey hasn't been changed

Text types in the wrong window

  • In terminal mode, make sure your terminal is the active focused window when you release the hotkey
  • Switch to clipboard mode and paste manually if auto-type is unreliable

TTS doesn't play anything

  • Check that Mute is not enabled (main window or tray icon)
  • If using Edge TTS, verify you have an internet connection
  • If using Piper, verify the voices\ and piper_bin\ folders are next to the exe
  • Check your Windows default media player can open .mp3 and .wav files

Claude's responses aren't being read

  • The Claude Code Stop hook must be active. On first run it's installed automatically — restart the app to trigger re-installation if needed
  • Check ~\.claude\settings.json — it should contain an entry with "event": "Stop" and voice_assistant_stop_hook in the command
  • Make sure the Voice Assistant app is running before you start a Claude Code session

Whisper model is slow to transcribe

  • Switch to the tiny or base model in Settings → Input → Whisper model
  • The first transcription after launch is slower (model loading). Subsequent ones are faster

Transcription is inaccurate

  • Speak clearly and at a normal pace
  • Switch to the medium model for better accuracy (uses more CPU)
  • Reduce background noise
  • Use the transcription preview to catch and cancel bad transcriptions before they're sent
