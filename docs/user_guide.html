Speak to Claude Code. Hear it speak back. Hands-free AI on Windows.

📦 Installation

The app is pre-installed at:

C:\Users\limfy\AppData\Local\VoiceAssistant\VoiceAssistant.exe

To create a desktop shortcut: open File Explorer, navigate to the folder above, right-click VoiceAssistant.exe → Send to → Desktop (create shortcut).

Note: The voices\ and piper_bin\ folders must stay in the same folder as VoiceAssistant.exe for offline TTS to work.

🚀 Launching the App

1
Double-click VoiceAssistant.exe from the desktop shortcut or the AppData folder.
2
A window titled "Voice Assistant" opens showing status: Idle.
3
A blue icon appears in the system tray (bottom-right, click ^ to show hidden icons).
4
On first run, the Claude Code Stop hook is installed automatically — no manual setup needed.

To close the main window without quitting: click X — the app keeps running in the tray. To fully quit: right-click the tray icon → Quit.

🗣️ Speaking to Claude

Uses toggle mode — press once to start, press again to stop. No holding required.

1
Open a terminal with a Claude Code session running (claude) and click inside it.
2
Press Ctrl+Shift+R — status changes to Recording… and your microphone activates.
3
Speak your question clearly.
4
Press Ctrl+Shift+R again — status changes to Transcribing… (takes 5–15 seconds).
5
A preview overlay appears at the bottom of your screen showing the transcription. Auto-sends in 2 seconds, or click Send / Cancel.
6
The text is delivered based on your Input Mode:
  • Terminal mode — auto-types into your active terminal (click the terminal first, then press Enter to submit to Claude)
  • Clipboard mode — copies to clipboard, press Ctrl+V to paste anywhere
In both modes, the text is also always copied to clipboard as a backup.
7
The transcription appears in the main window under Last transcription — click on it to select and copy manually.
Tip: The first transcription after launch takes longer (~15s) as Whisper loads into memory. Subsequent ones are faster (~3–8s).

🔊 Hearing Claude's Responses

When Claude Code finishes responding, the app captures the response text via a background hook.

On-Demand (default)

Press Alt+R to hear the last response read aloud. Use this when you want to listen at your own pace.

Auto-Play

Enable Auto-play TTS in Settings → Output to hear every response automatically.

What gets read

Only conversational prose — explanations, summaries, answers. Automatically skipped: code blocks, URLs, markdown headers.

Muting

Click Mute in the main window, or right-click the tray icon → Mute TTS.

Hotkey format

Use keyboard library syntax: ctrl+shift+r, alt+r, ctrl+f9, etc.

Valid modifiers: ctrl, shift, alt, windows

Avoid: alt+space (Windows system menu), ctrl+c, ctrl+v (clipboard), ctrl+z (undo).

📋 Input Modes

Terminal Mode

Transcribed text is auto-typed into the active window after a short delay. Click your terminal before speaking so it has focus when the text arrives.

Press Enter after the text appears to submit it to Claude.

Clipboard Mode

Transcribed text is copied to clipboard. Press Ctrl+V to paste anywhere — terminal, Claude.ai, ChatGPT, any text field.

Tip: In both modes, text is always copied to clipboard as backup. You can always paste even if auto-type fails.

🔧 Troubleshooting

App won't start — DLL error

A required Anaconda DLL is missing. Check the HANDOFF.md for the rebuild procedure which bundles all needed DLLs.

Stuck at Recording / won't stop

  • Press Ctrl+Shift+R again to toggle off
  • If unresponsive: right-click tray icon → Quit, or Task Manager → End Task on VoiceAssistant.exe

Transcription is slow

  • First transcription after launch is slow (model loading into memory) — subsequent ones are faster
  • Switch to tiny or base model in Settings for faster (less accurate) results

Text doesn't appear in terminal

  • Make sure your terminal is the active focused window when you press Send
  • The text is always in your clipboard — press Ctrl+V as fallback

TTS doesn't play

  • Check Mute is not enabled (main window or tray)
  • For Edge TTS: verify internet connection
  • For Piper: verify voices\ and piper_bin\ folders are next to the exe

Claude responses aren't captured

  • The Voice Assistant must be running before you start Claude Code
  • Check ~\.claude\settings.json has a Stop hook entry with voice_assistant_stop_hook
  • Re-run the app — it re-installs the hook on startup if missing
